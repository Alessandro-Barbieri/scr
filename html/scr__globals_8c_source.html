41 /* There are three directories where SCR manages files: control, cache, and prefix.
42  *
43  * The control directory is a fixed location where a job records its state and reads files
44  * to interpret commands from the user. This directory is fixed (hard coded) so that
45  * scr utility scripts know where to look to read and write these files.
46  *
47  * The cache directory is where the job will cache its checkpoint files.
48  * This can be changed by the user (via SCR_CACHE_BASE) to target
49  * different devices (e.g. RAM disc vs. SSD). By default, it uses the same prefix as the
50  * control directory.
51  *
52  * The prefix directory is where the job will create checkpoint directories and flush
53  * checkpoint files to. Typically, this is on a parallel file system and is set via SCR_PREFIX.
54  * If SCR_PREFIX is not set, the current working directory of the running program is used.
55 */
